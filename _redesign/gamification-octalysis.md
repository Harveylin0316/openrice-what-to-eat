# 好康地圖 遊戲化設計 — 八角框架（Octalysis）黑帽策略

> **目標**：把「打開看看」變成「每天必開」。黑帽驅力（CD6 稀缺 / CD7 不確定 / CD8 損失）
> 負責拉新與促活，白帽驅力（CD2 成就 / CD4 所有權）負責長期留存。
> **鐵律**：不造假。稀缺必須是真的（次數真的有限、今日之星真的每天換），
> 假倒數/假限量 = 信任死刑 + 消保法風險。

## 2. G1 黑帽機制（純前端 localStorage，本輪實作 ✅）

### 2.1 每日抽選額度（CD6 稀缺 · Magnetic Caps）
- 每日 **10 次**抽選（含「換一個」），FAB 顯示剩餘次數 badge
- 用完：老虎機鎖住 →「今日 10 次用完啦，明天 0 點重置 🎲」＋導向清單繼續逛
- **只鎖拉霸不鎖工具**：地圖 / 搜尋 / 清單 / 迷你卡完全不受限
- 儲存：`rr_map_dice = {date, used}`，跨日自動重置

### 2.2 連續天數 Streak（CD8 損失）
- 連續開啟好康地圖天數 🔥，斷一天歸零
- **有牙齒的損失**：連續 ≥3 天 → 每日抽選 +2 次（10→12）
- 開啟時 toast：「🔥 連續 N 天逛好康地圖！」（≥2 天才顯示）
- 額度用完的提示順帶預告：「明天回來，連續天數會讓抽選變多」
- 儲存：`rr_map_streak = {last, days}`（v2 接 Supabase 跨裝置）

### 2.3 大吉時刻（CD7 變動獎勵 · variable-ratio）
- 每次抽選 **12% 機率**觸發「✨ 大吉」：金色 badge + 一次性閃光動效
- **大吉奉還 1 次抽選次數**（toast 告知）——稀缺與驚喜互相咬合
- 動效一次性播放，不違反「無 infinite 動畫」效能鐵律

### 2.4 今日之星（CD6+CD7 每日輪換）
- 每天以日期種子從好康店（booking_offer ∪ coupon）挑 1 間
- 金冠 👑 圖釘常駐地圖（不聚合）、卡片帶「🌟 今日之星」badge
- 文案言明稀缺：「每天換一間，明天就不是它了」

## 4. 誠實護欄（不可越線）

1. 次數限制真實執行，不做「假裝快用完」
2. 今日之星真的每天換（日期種子可驗證）
3. 打烊倒數用真實營業時間，永不虛構
4. 大吉機率固定 12%，不做「新手必中」暗操作（要做需明示）
5. 黑帽只包裝**娛樂層**（抽選），不阻斷**工具層**（找店/看券/訂位）
